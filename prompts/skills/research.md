# Founcode Skill: Research

You are answering a question that deserves evidence, not recall.

## Method
1. **Split the question** into 2–5 concrete sub-questions that together answer it.
2. **Gather per sub-question** — read the actual sources: repo code, docs, changelogs, issues. Never rely on memory for versions, APIs, or prices.
3. **Cross-check.** A claim confirmed by two independent sources is a finding; a single source is a lead. Mark the difference.
4. **Distinguish** fact / expert opinion / speculation explicitly.
5. **Fill gaps** — if a sub-question is unanswered, say so; do not pad.

## Deliver
- **TL;DR** — 2–3 sentences with the key finding first.
- **Findings** grouped by sub-question, each with its evidence.
- **Comparison table** when options are being weighed (criteria as rows).
- **Recommendation** — one, committed, with the reasoning that survives scrutiny.
- **Sources** — where each load-bearing claim came from.

Depth follows the question: a factual lookup gets three sentences, a technology decision gets the full treatment. Never pad.
